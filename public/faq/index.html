Frequently Asked Questions

Focused answers to the controls and behaviors most users ask about.

Getting Started

What does this optimizer calculate?

It searches armor, charm, decoration, and optional weapon-assisted set bonus combinations that satisfy your selected skill targets and then ranks valid results.

Which settings should I choose first?

Start with Charm Mode = Off, Allow alpha/gamma = On, all decorations selected, and near-miss off. Once you get a stable baseline, enable Suggest After and near-miss for guided expansion.

What is the difference between Charm Mode Off, Suggest After, and Owned?

Off disables charm post-processing. Suggest After adds theoretical RNG charm templates after builds are generated. Owned evaluates your imported owned charm list.

Solver Behavior

What does Allow Near-Miss Solver Results do?

It keeps builds that are slightly short on requested skills so completion modes can still evaluate them. Near-Miss Max Missing Points controls how short those builds may be.

Why do I get no results?

Common causes are strict targets, narrow decoration pools, or restrictive filters. Lower one or two targets, broaden decoration selections, or enable near-miss.

Can set or group skills be rolled on RNG charms?

No. Set and group skills are excluded from RNG charm suggestions. Suggested templates only use rollable skill types.

Weapon Set Bonus Roll

Do I need to enable a weapon set-bonus toggle?

No. The optimizer automatically checks relevant Artian weapon +1 set-piece coverage when your requested skills include set/group bonus targets. This is deterministic modeling only. It does not estimate roll probability.

What does Requires weapon set bonus roll mean?

It means that build fails at least one requirement if the weapon contribution is removed. The result depends on your selected weapon roll assumption.

Data, Sharing, and Performance

Do shared links preserve weapon set-bonus behavior?

Yes. Share links preserve requested targets and filters, and automatic weapon set-bonus inference runs from that same state.

How do I use Owned Charm Builder with the optimizer?

Create or import charms in Owned Charm Builder, then switch optimizer Charm Mode to Owned. The optimizer reads the same local browser storage data.

What performance settings are recommended?

Start near 60 to 80 percent of available threads and Max Results Per Thread around 25 to 50. Increase gradually while monitoring responsiveness.

Where does data come from and what is stored locally?

Data comes from MHDB Wilds API endpoints. The app caches normalized data locally for faster reloads, and owned charm data stays in your browser.

Does this tool calculate weapon damage or roll probabilities?

No. It is a build feasibility planner for skills, set bonuses, and completion paths. It does not simulate hunt DPS or roll probability.
