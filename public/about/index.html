What This Tool Does

MH Wilds Skill-First Gear Optimizer is a skill-first planner for Monster Hunter Wilds. You choose targets, then the solver searches armor, decorations, and charm paths that can satisfy those targets under your filters.

  • Supports armor skills plus Set Bonus and Group Bonus requirements.
  • Compares charm modes: Off, Suggest After (theoretical), and Owned charms.
  • Automatically checks Artian weapon set-bonus coverage (+1 set piece) for relevant targets.
  • Highlights dependency badges so you can spot risky assumptions quickly.

This tool is for build planning and feasibility. It does not calculate weapon damage rotations or RNG probability for rolls.

How A Run Works

  1. Add desired skills and levels in Desired Skills.
  2. Choose allowed decorations in Decoration Pool.
  3. Set Controls (armor rarity toggles, charm mode, threads, result limits).
  4. Run Optimize and review required skills, defense/resists, and leftover slots.
  5. Use badges and flex suggestions to decide whether a result is practical for your hunt.

Tools In This Site

Skill-First Gear Optimizer: Main build search page with solver controls, near-miss handling, charm evaluation, and result ranking.

Owned Charm Builder: Form UI for creating/editing your charm library, plus JSON import/export. Owned mode reads this same local data.

Interpreting Results

  • Score (With Charm): Overall ranking score after selected charm evaluation path.
  • Charm dependence: How strongly completion depends on charm help.
  • Requires weapon set bonus roll: Build fails constraints if weapon set piece is removed.
  • Requested Skills: Shows exact met/missing levels per target.

Weapon Set Bonus Modeling (Artian)

Use Allow Artian weapon set-bonus roll (auto) to let the optimizer infer relevant Artian weapon set coverage (+1 set piece) from your requested Set/Group bonus targets.

  • Deterministic only: no probability weighting is applied.
  • Turn it off to view armor-only feasibility without weapon-assisted set bonus.
  • If a result depends on weapon contribution, it is labeled in the result header.
  • When possible, the label includes the specific required set name.

Beginner Filter Setups

Starter Realistic

Use when: You want only directly craftable, no-theory baseline sets.

  • Charm Mode = Off
  • Allow α armor = On
  • Allow γ armor = On
  • Decoration Pool = All selected
  • Allow Near-Miss Solver Results = Off

Starter With Guidance

Use when: You want practical sets plus simple theoretical completion hints.

  • Charm Mode = Suggest After
  • Allow Near-Miss Solver Results = On
  • Near-Miss Max Missing Points = 2
  • Show meets targets without charm = On
  • Show meets targets with best charm = On
  • Hide HIGH charm dependence = On (optional, cleaner list)

Advanced Filter Setups

RNG Theorycraft

Use when: You want to explore high-ceiling builds with broader completion paths.

  • Charm Mode = Suggest After
  • Allow Near-Miss Solver Results = On
  • Near-Miss Max Missing Points = 4 to 6
  • Hide HIGH charm dependence = Off
  • Increase Charm Suggest Count to compare multiple template paths

Owned Charm Validation

Use when: You want realistic results strictly from your personal charm inventory.

  • Charm Mode = Owned
  • Import/maintain charms in Owned Charm Builder
  • Allow Near-Miss Solver Results = On (2 to 3 suggested)
  • Show meets targets without charm = On
  • Show meets targets with best charm = On

Weapon-Assisted Set Bonus Check

Use when: Your target set bonus is one piece short and weapon roll coverage matters.

  • Add the relevant Set/Group bonus skill target(s) in Desired Skills.
  • Run Optimize and check for weapon-roll dependency badge(s).
  • Review Requires weapon set bonus roll badge before committing a build

Performance Tips

  • Start near 60 to 80 percent of your available threads, then tune upward.
  • Keep Max Results Per Thread around 25 to 50 unless you need deeper tails.
  • Constrain skill targets and decoration pool before increasing thread count.
  • For large near-miss searches, begin at missing points 2 to 3, then widen only if needed.

Sharing, Cache, and Privacy

  • Copy-link stores current state in a versioned share payload for compatibility.
  • Weapon-roll toggle state is included in share links so recipients reproduce the same assumptions.
  • Data is fetched from MHDB and cached locally for faster reloads and fallback behavior.
  • Owned charm data is stored locally in your browser; it is not uploaded by this site.

Data Source and Scope

Build data comes from MHDB Wilds API endpoints for armor, armor sets, skills, decorations, and charms.

This project prioritizes feasibility and planning transparency over damage simulation. Use it to discover valid combinations and understand trade-offs, then validate final hunt performance in-game.
